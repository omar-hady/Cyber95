# Cross-Site Scripting (XSS) Attack Demo

Cross-Site Scripting (XSS) is one of the most common web vulnerabilities. It occurs when a web application includes untrusted data in a web page without proper validation or escaping.

## How XSS Works

XSS attacks happen when:
1. A web application takes user input
2. The input is not properly sanitized
3. The input is displayed back to users without escaping
4. Malicious scripts can be executed in other users' browsers

## Common XSS Payloads

Here are some common XSS payloads you can try:

### Basic Script Injection
```html
<script>alert('XSS')</script>
```

### Image Tag XSS
```html
<img src="x" onerror="alert('XSS')">
```

### Event Handler XSS
```html
<input onfocus="alert('XSS')" autofocus>
```

### JavaScript URL
```html
<a href="javascript:alert('XSS')">Click me</a>
```

## Prevention Methods

1. **Input Validation**: Validate all user input on the server side
2. **Output Encoding**: Properly encode output when displaying user data
3. **Content Security Policy (CSP)**: Implement CSP headers to prevent script execution
4. **HTTP-Only Cookies**: Use HTTP-only cookies to prevent JavaScript access

## Impact

XSS attacks can lead to:
- Session hijacking
- Defacement of websites
- Redirection to malicious sites
- Theft of sensitive information
- Installation of malware
